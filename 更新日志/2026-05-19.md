# 2026-05-19

- 新增 `/method <主题|all>` 斜杠指令，用于动态切换寻找 issue 的主题；`all` 会恢复默认方向。
- 新增 `/prefer <GitHub 仓库链接>` 一次性重点关注分析；生成 PR 计划后通过飞书通知，用户用 `/prefer approve <计划ID>` 批准后会优先进入现有 Codex CLI 提 PR 流程，支持 `/prefer reject <计划ID>` 和 `/prefer list`。
- 新增 CPA budget 查询缓存与节流。成功结果缓存 10 分钟，失败结果缓存 30 分钟，最小查询间隔 10 分钟。
- 减少 CPA budget 查询的无效 fallback；没有 `auth_index` 的 Codex auth 文件只按文件名查询一次，避免每个账号额外打一发必失败请求。
- 修复 CPA budget 查询误用 `remote-management.secret-key` 的 bcrypt 哈希作为明文管理密钥的问题；当 CPA management 返回 401/403 时立刻停止本轮查询，避免触发 CPA 的 IP 失败次数封禁。
- 新增通用 `usage` 预算解析，OpenAI / Anthropic 兼容接口返回 `usage`、`limits`、`remaining` 等字段时会归一化为 runtime 可判断的预算状态；相关配置已在 README 中说明。
- 修复通过 CPA `/api-call` 查询 Codex budget 时 `$TOKEN$` 可能替换为不适合 `wham/usage` 的 token 的问题；现在优先使用本地 Codex auth 文件中的 `access_token`，且不写入日志或预算快照。
- 新增仓库冷静期机制：如果 PR 被关闭且未合并，会暂时跳过同一仓库的新任务，避免连续打扰维护者；服务器当前配置为 7 天。
- 新增仓库信任度策略：如果某个仓库已记录合并 PR 少于 3 次，planner 和 Codex 会优先选择小范围、低风险 PR。
